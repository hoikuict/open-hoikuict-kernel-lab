# open-hoikuict 100人規模デモデータ

このディレクトリには、定員100人規模の保育園を想定したデモデータを収録しています。すべて架空の情報で、開発、画面確認、デモ、テストデータ設計のために利用できます。

## 重要

- すべてダミーデータです。
- 実在する園児・保護者・職員の情報は含めないでください。
- メールアドレスは `demo.open-hoikuict.example` を使用しています。
- 本番DBに投入しないでください。

## ディレクトリ

```text
demo_data/
  import_compatible/   # 既存インポート機能に合わせたCSV
  full/                # モデルに近い詳細CSV
  summary.json         # 件数、対象期間、注意事項
```

## seed スクリプトで投入

```bash
python scripts/seed_demo_100.py --wipe-all
```

`--wipe-all` は既存データを削除します。ローカル開発DB以外では使わないでください。

## 既存インポート機能で入れる場合

`import_compatible/` 以下のCSVを次の順で読み込みます。既存インポート画面で新規登録する想定のため、ID列は空欄にしています。

1. `classrooms.csv`
2. `families.csv`
3. `children.csv`
4. `parent_accounts.csv`
5. `parent_child_links.csv`

この方法では、登降園や日次連絡などの詳細データは入りません。

## 整合性の前提

- 登降園記録は、園児100人 × 対象日21日分で 2,100件です。
- 日次連絡は、連絡未提出のケースを含めているため 2,090件です。
- `full/` の `created_by` や `reviewed_by` には、ユーザーIDではなく画面表示用の担当者名を入れている列があります。
